# Colour Scheme

To ensure that colours meet accessibility guidelines, it is important that the following colours are used in combination with each other correctly. These colours are defined in the /styles/variables.css file.

## Light Mode:

Primary Blue: #11167b - Used for button backgrounds, pagination button selection, headings, anywhere else colour may be needed
Background: #f5f5f5 - Used for the background of pages
Grey border: #aeaeae - Used for the outline of input fields and cards
Shadow: #00000025 - Used for box shadow on elements
Text: #000000 - Used for direct use on page backgrounds
Contrast Text: #ffffff - Used for use on element backgrounds that may be darker

## Dark Mode:

Primary Blue: #7eb0db - Used for button backgrounds, pagination button selection, headings, anywhere else colour may be needed
Background: #1c1c1c - Used for the background of pages
Grey border: #cecece - Used for the outline of input fields and cards
Shadow: #5c5c5c35 - Used for box shadow on elements
Text: #ffffff - Used for direct use on page backgrounds
Contrast Text: #000000 - Used for use on element backgrounds that may be darker

# Typography

Default browser fonts should be used across the site. Browsers already provide accessible options for customising the font, like changing the style or contrast, for disabled users, so overriding this may be frustrating for them.
